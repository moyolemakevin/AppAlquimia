INFORME INTEGRAL DE GESTION DE CONFIGURACION – PROYECTO ALQUIMIA ESENCIAL
Fecha: 21 de septiembre de 2025
Elaborado por: Oficina de Gestion de Configuracion de Software (OGCS)

=====================================================================
0. RESUMEN EJECUTIVO
=====================================================================
El presente informe consolida la evidencia requerida para la sustentacion final del Proyecto Integrador en Gestion de Configuracion de Software (GCS). Se evalua el cumplimiento del Plan de Gestion de Configuracion (CMP), la identificacion y control de Items de Configuracion (IC), la administracion de versiones, ramas, etiquetas y releases, el control de cambios (CR/CCB), la contabilidad de estado (CSA), las auditorias FCA/PCA, la automatizacion CI/CD y la generacion de release notes y artefactos. La auditoria concluye que Alquimia Esencial sostiene practicas coherentes con las mejores guias (GitFlow + SemVer), mantiene trazabilidad end-to-end y conserva integridad de entregables. Se reconocen oportunidades de refuerzo en pruebas automatizadas, gestion de secretos y observabilidad.

=====================================================================
1. INTRODUCCION, ALCANCE Y METODOLOGIA
=====================================================================
1.1 Introduccion
    La plataforma Alquimia Esencial integra una aplicacion hibrida Angular 20 + Ionic 8, un backend PHP 8.2 sobre MariaDB 10.4 y una coleccion de artefactos documentales (CMP, CSA, CRs, release notes, auditorias). El objetivo de este proyecto integrador es demostrar de manera profesional la Gestion de Configuracion aplicada, evidenciando estandares, herramientas y entregables alineados a la unidad 4 del silabo.

1.2 Alcance de la auditoria
    • Repositorio GitHub "AppAlquimia" y espejo local.
    • Directorios src/ (frontend) y api/ (backend), scripts SQL, configuraciones, assets.
    • Documentos: CMP_v1.0.pdf, CR_Log.xlsx, CSA.xlsx, ReleaseNotes_v1.1.0.md, QA_Report_2025-09-21.md, Registro_PCA_FCA_2025-09-20.pdf, Documento integrador (10 paginas), Presentacion (PPT/PDF).
    • Automatizacion: workflow ci.yml, configuraciones Ionic/Node.

1.3 Metodologia aplicada
    • Revision documental y checklist base (CMP, CR, CSA, release notes, QA, PCA/FCA).
    • Inspeccion de repositorio (ramas, PRs, tags, historial, pipelines).
    • Entrevistas breves con roles claves (Owner, Release Manager, Auditor).
    • Pruebas funcionales dirigidas (TC-FCA-01..05) y verificacion fisica (hashes, script SQL).
    • Evaluacion de resultados contra los criterios de la unidad 4 (estandares, herramientas, entregables).

2. POLITICA Y PLAN DE GESTION (CMP)
=====================================================================
2.1 Plan de Gestion de Configuracion
    • Documento CMP_v1.0.pdf (2 paginas) resguarda alcance, politicas, roles y responsabilidades.

4. VERSIONADO, RAMAS Y PULL REQUESTS
=====================================================================
4.1 Estrategia de versionado
    • SemVer adoptado de manera consistente:
        – v1.0.0 (baseline)
        – v1.0.1 (hotfix)
        – v1.1.0 (release auditoria)
    • Documentacion de versionado incluida en release notes y doc integrador.

4.2 Gestion de ramas (GitFlow)
    • main – release desplegada.
    • develop – integracion continua.
    • release/* – estabilizacion previa a main.
    • feature/* – nuevas funcionalidades (ej.: feature/add-whatsapp-cta, feature/admin-image-field).
    • hotfix/* – correcciones urgentes (hotfix/cors-bootstrap).

4.3 Pull Requests destacados (periodo auditoria)

6. CONTROL DE CAMBIOS (CR) Y ACTAS CCB
=====================================================================
6.1 Proceso CR ? CCB
    1) Registro CR_####.md + issue GitHub.
    2) Analisis impacto (alcance, costo, riesgo, requisitos afectados).
    3) Reunión CCB (Owner, RM, Auditor, QA) – decision documentada en Acta CCB.
    4) Implementacion (PR vinculado, pruebas, doc actualizada).
    5) Cierre (actualizacion CR Log, CSA, release notes si aplica).

6.2 Ejemplos recientes

=====================================================================
9.1 Workflow GitHub Actions (ci.yml)
    • Pasos: setup-node ? install ? lint ? test (placeholder) ? build ? checksum ? upload artifacts ? notify.
